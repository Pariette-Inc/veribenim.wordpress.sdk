## Какво е Veribenim?

Veribenim е **най-комплексната платформа в света за управление на съгласието и защита на личните данни**, разработена от Pariette. Интегрирайте я лесно в сайтовете си на WordPress, за да осигурите пълна правна съответствие по отношение на събирането и обработката на лични данни на посетителите.

---

## 🎯 Защо Veribenim?

### 1. Най-комплексното решение за KVKK
Veribenim е проектирана да отговори на всички изисквания, установени от турската администрация за защита на данни (KVKK). В съответствие с член 6, данните, основани на съгласие, се събират и управляват в рамките на явното съгласие. Системата за одит записва всяко съгласие.

### 2. Напълно съответстваща на GDPR архитектура
За сайтове, работещи в Европа, GDPR член 7 (Доказателство за съгласие) и член 13 (Задължение за прозрачност) са напълно удовлетворени. Многоезичният банер за съгласие отговаря на изискванията на GDPR.

### 3. Автоматично внедряване на скрипт
Не е необходимо кодиране след инсталирането. Veribenim автоматично внедрява JavaScript пакета в WordPress етикетите `<head>`. Криптирана SSL/TLS комуникация е стандартна.

### 4. Дружелюбна архитектура за разработчици
PHP куки, REST API крайни точки и WordPress система за куки за безглавия WordPress и персонализирани интеграции. Кратки кодове и Gutenberg блокове дават право на нетехнически потребители.

### 5. Поддержка на Multisite & Enterprise
Независима поддержка на токена за всеки подсайт в мрежата на WordPress Multisite. Възможна е глобална или специфична за сайта конфигурация.

### 6. Одит и докладване в реално време
Всички решения за съгласие се записват в базата данни. Панелът на администратора показва с дата и IP адрес кои категории са приети/отхвърлени.

## 🚀 Как да инсталирам?

### Метод 1: От панела на администратора на WordPress

1. Влезте в контролния панел на WordPress
2. Отидете на **Приставки → Добавяне на нова**
3. Потърсете **"Veribenim"**
4. Кликнете на **Инсталиране → Активиране**

### Метод 2: Composer

```bash
composer require veribenim/wordpress
```

След това активирайте приставката от контролния панел на WordPress.

### Метод 3: Ръчно инсталиране

1. Изтеглете `veribenim-wordpress.zip` от [страницата на версиите](https://github.com/pariette/veribenim-wordpress/releases)
2. Качете го в `wp-content/plugins/`
3. Активирайте го от контролния панел

### Конфигурация след инсталирането

1. Отидете на **Настройки → Veribenim**
2. Копирайте токена на вашия сайт от [app.veribenim.com](https://app.veribenim.com) → Вашия сайт → Интеграция
3. Поставете токена в полето **Токен на сайта Veribenim**
4. Изберете предпочеления си език (Türkçe, English, Deutsch, Français, Español, Български, العربية)
5. Кликнете на **Запазване**

✅ Банерът на Veribenim ще се появи на всички вашите сайтове.

## ⚡ Бързо начало

```php
// Когато съгласието е приемано
add_action('veribenim_consent_accepted', function($preferences) {
    // $preferences = ['necessary'=>true, 'analytics'=>true, 'marketing'=>false]
    error_log('Съгласието е приемано: ' . json_encode($preferences));
});

// Когато съгласието е отхвърлено
add_action('veribenim_declined', function() {
    error_log('Съгласието е отхвърлено');
});

// Добавяне на метаданни към съгласието
add_filter('veribenim_consent_metadata', function($metadata) {
    $metadata['user_role'] = get_current_user()->roles[0] ?? 'guest';
    return $metadata;
});
```

## 🎨 Функции

- **Автоматично внедряване на скрипт** — JavaScript пакета автоматично внедрен в раздел `<head>`
- **Панел за настройки на администратора** — Токен, език и персонализация на банера в една страница
- **Дизайн независим от темата** — Работи със всички теми на WordPress
- **Съвместим с WooCommerce** — Съгласие в касата, съхранено с детали на поръчката
- **AJAX-базирани актуализации** — Актуализирайте предпочитанията без презареждане на страница
- **REST API поддержка** — Интеграция на безглавия WordPress
- **Краткия кодекс и Gutenberg блок** — Поддержка на краткия кодекс `[veribenim_consent]` и блок редактор
- **Съвместим с приставки за кеш** — Работи с WP Rocket, W3 Total Cache, LiteSpeed Cache
- **Поддержка Multisite** — Активиране на мрежата и конфигурация специфична за сайта

На сайтове WooCommerce:

- Преглед на детайлите за съгласието при **WooCommerce → Поръчки → [Поръчка] → Съгласие Veribenim**

## 👨‍💻 Куки за разработчици

```php
// Преди зареждането на банера за съгласие
add_action('veribenim_before_load', function() { });

// След приемането на съгласието
add_action('veribenim_after_consent', function($preferences) { });

// Добавяне на персонализирани категории съгласие
add_filter('veribenim_consent_categories', function($categories) {
    $categories['custom'] = ['label' => 'Персонализирана', 'description' => 'Описание'];
    return $categories;
});
```

**REST API крайни точки:**
```bash
GET /wp-json/veribenim/v1/consent/{user_id}
POST /wp-json/veribenim/v1/consent
```

## 🔒 Стандарти за безопасност

### KVKK съответствие

- ✅ **Член 6 :** Обработка на базата на съгласие
- ✅ **Член 10 :** Права на субекта на данни
- ✅ **Член 13 :** Задължение за прозрачност
- ✅ **Член 14 :** Безопасност на данните

### GDPR съответствие

- ✅ **Член 4 :** Проектирана като платформа за управление на съгласието
- ✅ **Член 7 :** Свободно дадено, специфично и информирано съгласие
- ✅ **Член 13 :** Информация за субекта на данни
- ✅ **Член 17 :** Право на забравяне (Право на изтриване)
- ✅ **Член 32 :** Безопасност на обработката

### Техническа безопасност

- **Криптиране :** TLS 1.3 (AES-256-GCM), база данни AES-256
- **Комуникация на API :** HTTPS/TLS 1.3, Bearer токен, CORS, ограничение на скоростта
- **Одит :** Времева печат, IP адрес, User-Agent, дневник на изменения
- **Сертификати :** ISO 27001, SOC 2 Type II

### Задържане на данни

- Необходими бисквитки: Правно задължение (неопределено)
- Аналитика: 30 дни
- Маркетинг: 90 дни
- Дневник на одитите: 1 година

## ❓ Често задавани въпроси

**В: Мога ли да персонализирам Veribenim според моите нужди?**

О: Да! Персонализирайте цветовете, позицията и текста на банера от панела на администратора. Разширеното персонализиране използва PHP куки и филтри.

---

**В: Защитава ли Veribenim данните на клиентите в WooCommerce?**

О: Да, Veribenim автоматично показва изисквания за съгласие при касата на WooCommerce и съхранява предпочитанията с поръчката.

---

**В: Влияе ли Veribenim на моя SEO?**

О: Не. Veribenim се зарежда асинхронно и не влияе негативно на скоростта на страницата или резултатите на Google PageSpeed.

---

**В: Мога ли да използвам различни токени за всеки сайт в WordPress Multisite?**

О: Да, всеки подсайт може да има собствен токен или да използва глобалния токен на администратора на мрежата.

---

**В: Какви данни съхранява Veribenim?**

О: Само предпочитания за съгласие, времева печат и IP адрес. Чувствителните лични данни (кредитни карти, номер на социално осигуряване и т.н.) никога не се съхраняват.

---

**В: Мога ли да тестирам Veribenim преди да се ангажирам?**

О: Да, предлагаме безплатен 30-дневен пробен период. Регистрирайте се на [app.veribenim.com](https://app.veribenim.com).
